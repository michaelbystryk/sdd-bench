## Why P6 first
Cheapest task to author, most likely to differentiate, no answer key to seal. It exercises
every pipeline stage except the planted-truth recall/precision path (which P1/P8/P9/P10
add later). Run end-to-end on two arms, it validates the harness before the 44-cell scale-up.

## Pipeline decisions locked by the pilot

1. **Cells do NOT run as Workflow subagents.** A blindness probe confirmed a Workflow
   `agent()` runs in the harness repo cwd and inherits the harness `CLAUDE.md` — it
   immediately self-identified as being inside "the eval harness" on the party-track and
   could read the rubric/answer-keys. That **fully unblinds a cell** (violates the
   load-bearing blindness rule), beyond the milder "run-harness asymmetry" the brief named.
2. **A1/A2/A3 run blind via headless `claude -p`** through a new `cell-headless.sh party`
   subcommand: brief (+ arm wrapper) inline, `reference/` seeded, executed in a throwaway
   `mktemp` dir with no harness `CLAUDE.md`. Deliverable captured as the written file; cost
   from the result JSON (`total_cost_usd` + per-model `modelUsage` + `duration_api_ms`) —
   authoritative, no `/status`, no manual pricing math. The wrapper was already
   operator-authorized for the main-track automated arm, so no new permission grant.
3. **Workflow is the right vehicle for blind *scoring*** (not cell execution): N raters
   pinned to `model:'fable'` (Mythos) for the cross-model circularity break, each reading
   one self-contained, arm-free context file (brief + reference + rubric + guidance +
   scrubbed deliverable). Structured output → no parsing. 2 raters/output landed
   **within 1 pt** on both — the rubric's design goal.
4. **A4 stays manual** (`runs/party/p06-quick-decision/a4/run-001/RUNBOOK.md`) and runs
   FIRST in the real ordering (sets A2's `MAX_THINKING_TOKENS`). A1/A3 were run ahead only
   to validate the apparatus — independent blind dirs, no contamination.

Two new harness pieces, both committed: `cell-headless.sh party` and
`harness/party/scripts/scrub.sh` (scan → safe-strip → re-scan → sha256 → sealed arm↔label map).

### Two signals, both pointing the masquerade way — on this task, so far

- **Persona framing left no detectable trace.** All four raters guessed **A2** for *both*
  outputs and explicitly reasoned *away* from a persona arm ("no roundtable residue, reads
  like one model thinking long"). A3's roundtable *did* genuinely disagree in the working

  the synthesized deliverable is **indistinguishable from solo**. Persona machinery you
  can't detect in the output is the masquerade thesis in miniature.
- **The extra spend bought cost, not quality.** A3 cost 1.47× for −0.5 pt of quality, and
  its one persona-flavoured flourish — an invented "agents scan 20–40 rows" stat, untagged —
  was the *only* correctness dock in the four ratings. More voices, one more phantom claim:
  the precision-erosion the planted-truth tasks are built to measure, showing up here on the
  rubric's correctness dim instead.

## Hard caveats (why this is a partial read)
- **n=1 per arm; A4 and A2 not yet run.** A3 (single-prompt roundtable) is *not* the
  machinery — A4 is. The real masquerade ladder (A4 vs A3 vs A2 vs A1) needs all four. A3≈A1
  here only says the *persona-prompt* replica adds no quality on a small task.
- **Composite divisor not locked.** The P6 cost-tax divisor locks only after A4's heavy
  spend anchors the top end (overlay § C). With the placeholder $0.05 divisor the tax is
  tiny; A4 (main track saw 15–172× for BMAD) is what will actually flip the composite, if
  the over-ceremony thesis holds here.
- **Quality clusters at the cap.** Both arms exceed the P6 right-sizing cap (Q_capped 20),
  so on this task quality is already a wash and **cost is the whole game** — exactly the
  condition P6 was designed to create. Expect the cross-arm story to live in the cost
  column once A4 lands.
